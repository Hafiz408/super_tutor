**Setup (do once):** Load or create a study session with a quiz. Make sure both backend (port 8000) and frontend (port 3000) are running.

**Scenario 1 — QUIZ-04 + ADVISE-01: CASE 5 Proactive Advisor auto-fire**
1. Open the Tutor tab and ask the tutor to quiz you: type "quiz me" and send.
2. Answer 2-3 questions INCORRECTLY (pick the wrong letter deliberately).
3. Do NOT ask "what should I focus on?" — wait for the coordinator to auto-fire the Advisor.
4. PASS: The tutor's next response mentions a specific weak concept or focus area without prompting.
5. FAIL: Tutor says "That's incorrect, the answer is X" with no Advisor proactive output — note in response.

**Scenario 2 — ADVISE-01 + TEAM-07: CASE 6 Explicit Advisor**
1. In the Tutor tab, type "what should I focus on?" and send.
2. PASS: Advisor response names at least one specific concept from the session material as a focus area.
3. FAIL: Generic response with no named concept.

**Scenario 3 — ADVISE-02: CASE 7 ContentWriter on Advisor acceptance**
1. After the Advisor surfaces a named focus area in Scenario 1 or 2, type "yes" and send.
2. PASS: The tutor generates targeted inline content (extra flashcards or a mini-quiz) on the named concept. Content renders formatted in the chat bubble (not raw markdown syntax).
3. FAIL: "yes" triggers a generic response, or ContentWriter generates content unrelated to the focus area.
   Note: CASE 7 only fires when the PRIOR assistant turn was an Advisor suggestion. If other messages were sent between the Advisor turn and "yes", CASE 7 may not fire — expected per STATE.md.

**Scenario 4 — ADVISE-03: Quiz score persistence round-trip**
1. Complete the session quiz (answer all questions), click "See results →".
2. Check backend logs — should see `Adaptive data persisted — session_id=...`.
3. Refresh the browser page (Cmd+R or F5).
4. Open DevTools Network tab → find the `GET /sessions/{session_id}` request → inspect the response.
5. PASS: Response contains `"tutor_adaptive_data": {"quiz_score": {"correct": N, "total": M}, "focus_areas": [...]}` with actual scores from Step 1.
6. FAIL: `tutor_adaptive_data` is missing or `{}` after refresh.

**Scenario 5 — End-to-end regression**
1. Open a fresh browser window (Incognito if needed).
2. Complete the full flow: wrong answers → CASE 5 Advisor fires → type "yes" → ContentWriter delivers targeted content → complete quiz → refresh → verify quiz score in session GET.
3. PASS: All steps complete without errors, streamed responses appear in chat bubbles.
