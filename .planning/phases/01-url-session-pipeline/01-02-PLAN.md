**OAT UI CSS integration:**
OAT UI (oat.ink) is a vanilla HTML/CSS zero-dependency library. It styles semantic HTML elements directly by tag and ARIA attribute — not React components. Import it via CDN link in the root layout `<head>`, OR install via npm if a package is available.

Check: `npm info oat-ui 2>/dev/null` — if a package exists, install it and import in globals.css. If not, use the CDN link.

In `frontend/src/app/layout.tsx`, include OAT UI via `<link>` in the metadata approach or inline in the `<head>`:
```typescript
import type { Metadata } from "next";
import "./globals.css";

export const metadata: Metadata = {
  title: "Super Tutor",
  description: "Turn any article into a complete study session",
};

export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="en">
      <head>
        <link rel="stylesheet" href="https://oat.ink/oat.css" />
      </head>
      <body>{children}</body>
    </html>
  );
}
```

If OAT UI does not have a CDN link at that URL, check oat.ink for the correct CDN URL or npm package name and use whichever is available. If neither works, leave a clear TODO comment in globals.css and continue — the functional logic is more important than the exact CSS source for this plan.

**frontend/src/types/session.ts** — TypeScript types mirroring backend Pydantic models exactly (same field names, same shapes):
```typescript
export type TutoringType = "micro_learning" | "teaching_a_kid" | "advanced";

export interface SessionRequest {
  url?: string;
  paste_text?: string;
  tutoring_type: TutoringType;
  focus_prompt?: string;
}

export interface Flashcard {
  front: string;
  back: string;
}

export interface QuizQuestion {
  question: string;
  options: string[]; // exactly 4
  answer_index: number; // 0-3
}

export interface SessionResult {
  session_id: string;
  source_title: string;
  tutoring_type: TutoringType;
  notes: string; // markdown
  flashcards: Flashcard[];
  quiz: QuizQuestion[];
}

// SSE event shapes from the backend stream
export interface ProgressEvent {
  message: string;
}

export interface CompleteEvent {
  session_id: string;
}

export interface ErrorEvent {
  kind: "paywall" | "invalid_url" | "empty" | "unreachable";
}
```

Also add `SSE_STEPS` constant for progress bar (used by loading page):
```typescript
export const SSE_STEPS = [

  "Crafting your notes...",
  "Making your flashcards...",
  "Building your quiz...",
] as const;
```
